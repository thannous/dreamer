# Audit SEO complet - Noctalia `docs/`

**Date :** 2026-04-29  
**Portee :** site statique genere dans `docs/`  
**Domaine :** `https://noctalia.app`  
**Contexte :** site marketing + pSEO multilingue EN/FR/ES/DE/IT pour l'app de journal et interpretation de reves Noctalia.

## Resume executif

Le socle SEO technique est sain : `npm run docs:check` passe, le sitemap contient 620 URL canoniques, aucun lien interne casse n'a ete detecte, les canonicals/hreflang sont coherents, et toutes les pages indexables ont title, meta description, H1 et JSON-LD.

Les priorites ne sont plus la crawlabilite de base. Les freins principaux sont :

1. **FAQ JSON-LD non alignee avec le contenu visible** sur 30 questions.
2. **Liens externes scientifiques casses** : 10 URL externes en 404.
3. **Titles et meta descriptions trop longs** sur plusieurs articles recents, surtout DE/ES/FR/IT.
4. **Charge JS/images sur les homes** : `lucide.min.js`, GSAP et hero images peuvent peser sur LCP/INP.
5. **Donnees Search Console non accessibles via MCP actuel** : il faut soit exporter depuis l'UI, soit connecter l'API Search Console via OAuth/service account autorise.

## Architecture et indexation

### Points forts

- `robots.txt` autorise le crawl et declare le sitemap.
- `sitemap.xml` contient 620 URL, sans URL `.html`.
- Aucun URL du sitemap ne pointe vers un fichier local manquant.
- Aucune page indexable n'est absente du sitemap.
- Aucun canonical duplique detecte.
- `docs/_redirects` normalise HTTP, www, `.html`, anciens slugs FR et pages legales legacy.
- `docs/_headers` noindex les zones utilitaires, templates, rapports, scripts, markdown internes et pages d'auth.

### Point a surveiller

La page anglaise `/en/` existe en fichier local mais est `noindex` et redirigee vers `/`. C'est coherent si `/` est la home anglaise canonique. Par contre, 124 pages indexables contiennent encore des liens UI vers `/en/`. Ce n'est pas bloquant, car `/en/` redirige, mais c'est un signal interne moins propre que des liens directs vers `/`.

**Recommandation :** remplacer les liens internes de selection de langue anglaise vers `/` dans les pages indexables, ou confirmer que le redirect `/en/ -> /` est volontaire et durable.

## International SEO

- Le build valide 620 pages canoniques avec hreflang.

  - root : 1
  - EN : 123
  - FR : 124
  - ES : 124

### Risque

Le choix `/` comme canonical EN et `/en/` comme variante locale redirigee est acceptable, mais il faut eviter les signaux mixtes :

- Dans les pages indexables, l'alternate `en` devrait pointer vers `/` quand le cluster utilise `/` comme URL anglaise canonique.
- Les liens de navigation vers l'anglais devraient pointer vers `/`, pas `/en/`.

## On-page SEO

### Titles

Toutes les pages indexables ont un title unique. Aucun duplicate title detecte.

Problemes :

- Plusieurs titles sont trop longs pour un snippet SERP fiable, notamment :
  - `es/blog/alteracion-sueno-primavera-suenos.html` : 145 caracteres.
  - `de/blog/fruehling-schlaf-stoerung-traeume.html` : 118 caracteres.
  - `es/blog/deuda-sueno-salud-suenos.html` : 116 caracteres.

Il existe aussi des titles courts sur pages legales ou utilitaires, mais ce n'est pas prioritaire SEO.

**Recommandation :** ramener les articles acquisition entre 45 et 65 caracteres, avec la requete principale en debut de title.

### Meta descriptions

Problemes :

### H1 et contenu

## Performance SEO

Signaux locaux :

- `docs/js/lucide.min.js` : 386 KB.
- `docs/js/gsap.min.js` : 72 KB.
- `docs/js/ScrollTrigger.min.js` : 43 KB.
- `docs/img/hero/noctalia-observatory-bg.png` : 1,82 MB.
- `docs/img/img_presentation.png` : 1,26 MB.
- Home : plusieurs scripts defer + module `landing-animations.js`.
- Les images principales utilisent `srcset`, dimensions, `fetchpriority="high"` et lazy loading sur les images secondaires.

Risque :

- Les homes peuvent souffrir en LCP si le hero ou screenshot 1600w est trop lourd sur mobile.
- Lucide charge toute la librairie alors que chaque page n'utilise qu'un sous-ensemble d'icones.
- GSAP/ScrollTrigger ne devraient charger que la ou l'animation est utile.

**Recommandation :**

1. Convertir les gros PNG hero en AVIF/WebP avec fallback.
2. Auditer PageSpeed Insights sur `/`, `/fr/`, `/es/blog/suenos-de-muerte`, `/en/blog/precognitive-dreams-science`.
3. Remplacer `lucide.min.js` global par un bundle d'icones minimal ou des SVG inline generes au build.
4. Charger GSAP uniquement sur les pages qui en ont besoin.

## Marche a suivre pour recuperer les donnees Search Console

### Option rapide : export manuel

1. Ouvrir l'URL Search Console avec le compte qui a acces a `sc-domain:noctalia.app`.
2. Periode : les 3 derniers mois.
3. Exporter en CSV ou Google Sheets.
4. Exporter au minimum ces onglets :
   - Graphique / dates.
   - Requetes.
   - Pages.
   - Pays.
   - Appareils.
5. Placer les fichiers dans `docs/audits/search-console-2026-04-29/`.
6. Relancer l'analyse sur les CSV.

### Option propre : API Search Console

Utiliser l'endpoint :

```http
POST https://www.googleapis.com/webmasters/v3/sites/sc-domain:noctalia.app/searchAnalytics/query
```

Scope OAuth minimal :

```text
https://www.googleapis.com/auth/webmasters.readonly
```

Requetes utiles :

```json
{
  "startDate": "2026-01-29",
  "endDate": "2026-04-27",
  "dimensions": ["date"],
  "type": "web",
  "rowLimit": 25000
}
```

```json
{
  "startDate": "2026-01-29",
  "endDate": "2026-04-27",
  "dimensions": ["query"],
  "type": "web",
  "rowLimit": 25000
}
```

```json
{
  "startDate": "2026-01-29",
  "endDate": "2026-04-27",
  "dimensions": ["page"],
  "type": "web",
  "rowLimit": 25000
}
```

Ajouter aussi `["country"]`, `["device"]`, puis `["page", "query"]` pour croiser pages et requetes. Pour plus de 25 000 lignes, paginer avec `startRow`.

### Option MCP recommandee

Il faudrait ajouter un serveur MCP dedie Search Console ou un MCP generic Google APIs avec OAuth et scope `webmasters.readonly`. Le serveur doit exposer au minimum :

- `searchAnalytics.query(siteUrl, startDate, endDate, dimensions, rowLimit, startRow)`
- `sites.list()`
- `sitemaps.list(siteUrl)`
- optionnel : `urlInspection.index.inspect`

Sans ce MCP, l'export manuel reste la voie la plus rapide.
